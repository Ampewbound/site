So you wanna help develop the site?

Languages

  1. First, obviously, you're gonna want to learn HTML, which is, basically, what makes websites. It's definently the easiest of the three, and I reccomend that you get started over at W3Schools (the creators of HTML) or over at CodeCademy. I used CodeCademy's tutorial, but I'd bet W3Schools is a bit better. If niether of those options (for some reason) suit you, there's plenty more with a quick search. But, trust me, it is not hard at all.
  2. Second, you're going to want to learn CSS. It's what makes websites look like... websites. CSS is what is making the font of this page Calibri, the background black, the border grey and the text white, as well as all of the spacing and things like italics. The earlier mention of CodeCademy's tutorial includes CSS, and W3Schools has one as well (since, after all, they did create it).
  3. (Optional, for now) Thirdly, JavaScript is what makes things look fancy and sleek. This is not required, but you might want to go take a gander at it. From what I can gather, W3Schools's tutorial appears far superior to CodeCademy's, but this is entirely up to preference.

GitHub

  1. Obviously this site is hosted on GitHub. Projects on GitHub are in repositories. In order to actually contribute, you must first fork, or copy, the repository to your own account.
  2. Second, you need to submit a pull request (a request for the pushing of your update) once you make your desired edit. The guidelines for pull requests for this site are as follows.
    • Try to only submit a single change in each pull request. It does not matter the size of the change, but simply what you change.
    • Try to accurately name the request based on your changes, followed by the date and your name. For example:

      BodyOverhaul_8-23-16_Landon or MarginAdjustments_8-26-16_Logan
  3. Send the request and you are finished! Your request will be reviewed and, if all is liked and accepted, will be pushed to the main site. Sometimes you will find that one of your updates made it through but not the pull request that made it. That would be because one change was like, and another wasn't. This is why you should only put one change in one pull request. Also, be sure that I know what username you are operating under.

Thanks for reading! Remember, anything in this list (guidelines, instructions, ect.) are subject to change. I will try to notify you when it is updated.
